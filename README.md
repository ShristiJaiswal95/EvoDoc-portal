# EvoDoc Frontend Portal

## Overview

This project is a frontend implementation of a healthcare portal designed to manage patient intake, appointment booking, and basic doctor workflows. The focus was on creating a simple and usable interface for real-world clinical usage.

## Tech Stack

* CSS

---

## Features

* Patient intake form
* Appointment booking system
* Appointment list view
* Doctor dashboard
* Patient details and notes

---

## Key Decisions

* Focused on core healthcare workflows instead of adding unnecessary features
* Used plain HTML, CSS, and JavaScript for simplicity and faster development
* Prioritized usability and clarity over complex design

---

## Trade-offs

* No backend integration (data is not stored permanently)
* Basic validation and logic implemented for demonstration
* UI kept simple due to time constraints
